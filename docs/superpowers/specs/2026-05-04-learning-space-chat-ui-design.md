# 学习空间聊天式重构设计

## 目标

把当前偏“功能堆叠”的学习页，重构成一个更统一的“学习空间”产品形态。

核心变化：

- “学习目标”对外统一改名为“学习空间”。
- 主工作台改成 ChatGPT / DeepSeek 风格的聊天页面。
- 学习模式切换上移到聊天区顶部，成为主操作之一。
- 章节、资料、统计、测验保留，但降级为学习空间内的辅助入口。

## 命名

建议前台统一使用“学习空间”。

原因：

- 比“学习目标”更贴合用户心智，用户更容易理解是在围绕一门课程持续学习。
- 能自然容纳“英语 / Java / 网络”等不同主题。
- 也更适合承载“整体学习”和“按知识点学习”两种路径。

## 页面结构

### 主页面

主页面采用三栏式聊天工作台：

- 左侧：学习空间列表 + 当前空间知识点树 + 快速入口。
- 中间：聊天主区域，完整承担 AI 对话。
- 右侧：模式切换、当前知识点状态、历史记录、资料、学习进度。

### 顶部区域

聊天页顶部固定一个控制条：

- 当前学习空间选择
- 学习模式切换
- 当前知识点 / 当前进度
- 进入资料、统计、测验的快捷按钮

### 主聊天区

聊天区保持网页版 AI 产品的基本形态：

- 消息流居中展示
- 输入框固定在底部
- 支持 Markdown 渲染
- 支持流式输出

## 信息架构

### 一级入口

只保留一个强主入口：

- 学习空间

### 二级入口

放到学习空间内部：

- 知识点
- 资料
- 学习统计
- 测验
- 历史会话

## 组件调整

- `MainLayout`：从“传统后台侧边栏 + 内容区”改为更轻的应用框架，弱化全局导航存在感。
- `HomePage`：改为学习空间工作台首页，主视觉是聊天区。
- `LearningGoalsPage`：改名与改文案为学习空间列表页。
- `LearningSessionPage`：重构为新的聊天主页面，模式切换上移，左侧信息栏收窄。
- `ChatBox`：保留为消息流与输入核心组件，但外层样式改成更像 ChatGPT 的会话面板。

## 非目标

- 不在本次重构里改后端接口字段名。
- 不强制合并所有旧页面，只改前台体验与导航组织。
- 不做品牌级视觉识别重做，只先完成布局和信息架构重构。

## 风险与约束

- 现有页面很多依赖“学习目标”命名，前台改名后要保证导航和空状态文案统一。
- 聊天式布局需要兼顾桌面端和窄屏，右侧信息栏在移动端应自动折叠为抽屉或下沉区。
- 现有会话页逻辑较重，重构时要避免影响 SSE 流式消息和会话恢复。

## 验收标准

- 用户进入后看到的是“学习空间 + 聊天”而不是传统管理后台。
- 能清楚地在顶部切换学习模式。
- 能从一个学习空间内完成“整体学习”和“按知识点学习”。
- 聊天页视觉上接近现代 AI 聊天产品，而不是表单页或后台页。
